# Comments

Comments must be **extremely** concise. Most things can be conveyed in a single, short, terse sentence. That is the default; no long-winded LLM rubbish.

Before you write a comment, think about whether it would make sense to a human who is skim reading the code.

After you write a comment, pause to think about it again; is this illogical? Could I write this more clearly? Does it need the detail that is there, or would it actually be better if it were one shorter, high-level sentence?
